SEO with JiH + SSI

JiH (JSX-in-HTML), where pages are just plain HTML that renders JSX component, enables simple SEO and server-side includes with meta[href] props.

<head>
   <title>Home</title>
   <meta name="description" content="Home page" />
   <meta href='_metadatas.html' />
</head>

<h1>Home</h1>

<Panel/> <!-- invoking a JSX component -->

server components

The React server components are fully supported with function serialization and no "use client" or "use server" directives. The client-side API is represented in server-side just interfaces that will be run in client-side.

async function AwaitMe() { ... }

As cleaner Suspense replacer, the async props show the styled container element as loading replacer until a async component is loaded, that it changed to its children.

const Sample = props => <>
   <progress async>
      <AwaitMe />
   </progress>
</>

Full server-side rendering with dynamic, static and incremental generation:

SSR description
static generated HTML file during build time
dynamic HTML content in each request (default)
periodic intervalled HTML file rebuild 
import { static, dynamic, periodic } from '@reactive'

@static
export async function About() { /*...*/ } 

@dynamic
export async function Users() { /*...*/ } 

@periodic({ time: 1, unit: "day" }) 
export async function Newsletter() { /*...*/ }

dependency injections

IoC container is enabled by inject fluent method between server and render call.

React.server(3000, "routes")
     .inject(/* overloads */)
     .render(...)

Inject methods has multiple overloads that cover multiple innovative and advanced features, allowing fully configurability with simple APIs.

resource interface description
request (Request) => Response request interceptor
property (props: any, PropertyArgs) => any custom props injection
plugin (path:string, code:string) => string transpiler Deno plugin
setup DenoConfiguration Deno configuration object
parser RenderingHandler render interceptor

props injection

Property injection brings attribute directive to React using very similar implementation of functional components.

const show = props => ({ ...props,  hidden: !props.show  })

React.server(3000, "/routes")
     .inject(show)
     .render(App)

const App = props => <div show={true}>shown!</div>

Module declaration is needed to enable intellisense in TypeScript.

declare module "react" { 
   interface HTMLAttributes<T> { show?: boolean } 
}

As second argument, PropertyHandler offers some extra advanced features.

interface PropertyArgs { 
   uid:number,  // DOM identifier
   tag:string,  // current tag name
   ioc:any      // IoC container object
}

The ioc object allows contextual dependency injection in component, like in invalids object of reactive forms.

function SampleOfDI(props, { invalids }) {
   return <form data={props}> ...etc </form>
}

plugins resolvers

Plugin injection is a simplified API to add plugins into reactive inner Deno settings, allowing any new transpiler resource compatible with Deno.

type CodeHandler = (path:string, code: string) => string

Some of inner reactive resource are Deno plugins, like modular CSS, function decorators, etc. Resource that could expandis semantics in JSX.

rendering parsers

Reactive parser is a very special instropection to React rendering proccess. It is used for server-side rendering and props injection feature, but also could be handle by any developer, injecting an ParsingHandler instance.

interface RenderingHandler {
   onComponent(node: ReactNode): ReactNode
   onElement(node: ReactNode): RactNode
   onProps(props: any): any
}

build options

The default Deno build configuration could be changed or extended

 await esbuild.build({
   write: false,
   bundle: true,
   format: "esm",
   absWorkingDir: Deno.cwd(),
   plugins: [...denoPlugins()],
   outfile: "./builds/esm.js",
   entryPoints: ["./index.tsx"],
});

The overload is based in Deno BuildOptions insterface.

export interface BuildOptions extends CommonOptions {
  bundle?: boolean
  splitting?: boolean
  // etc....
}
